## 三、下一阶段计划

### 阶段 3：前端工程化

目标：将当前原生前端升级为可长期维护的工程化前端。

建议技术路线：

```text
Vite + React 或 Vue
组件化状态管理
路由与布局拆分
统一 API Client
统一设计令牌
```

任务：

- 拆分登录页、主工作台、侧栏、消息列表、线程面板、审计面板、成员面板。
- 抽象 Button、Input、Avatar、Badge、Modal、Panel、MessageBubble。
- 引入统一主题变量，与 `DESIGN_SPEC.md` 对齐。
- 增加前端构建脚本与静态资源输出目录。
- 增加基础页面级冒烟测试。

验收：

- 功能与当前 MVP 等价。
- 视觉与 Pencil 主画板一致。
- `npm run smoke` 继续通过。

### 阶段 4：企业安全与管理

目标：补齐私有化企业协同所需的基础管理能力。

任务：

- 管理后台。
- 用户管理。
- 角色权限配置。
- 频道管理。
- 审计详情页。
- 审计导出。
- 密码策略。
- 登录失败锁定。
- IP 白名单。
- 文件类型与大小策略。

验收：

- 管理员与普通用户权限边界清晰。
- 审计可按时间、操作者、类型过滤。
- 高风险操作有审计记录。

### 阶段 5：消息与协作增强

目标：提升日常协同效率。

任务：

- @ 人提醒。
- 频道公告。
- 置顶消息。
- 消息收藏。
- 图片预览。
- 文件列表。
- 搜索结果定位到消息上下文。
- 草稿。
- 表情回应。
- 在线状态优化。

验收：

- 频道协作流程更接近真实企业使用。
- 搜索、文件、线程能形成完整闭环。

### 阶段 6：生产级部署架构

目标：从 MVP 单机形态升级为可生产部署形态。

任务：

- PostgreSQL 适配。
- Redis 会话与 WebSocket 分布式广播。
- 对象存储适配。
- 反向代理配置。
- 备份与恢复。
- 健康检查与监控指标。
- 日志分级与审计日志落盘。
- 数据库迁移版本管理增强。

验收：

- 支持单机私有化部署。
- 支持后续平滑扩展到多实例。
- 数据备份与恢复流程可演练。

### 阶段 7：企业集成

目标：接入企业已有身份、组织和办公系统。

任务：

- 单点登录。
- LDAP / 企业微信 / 钉钉组织同步适配预留。
- Webhook。
- Bot 消息入口。
- 内部知识库关联。
- 审批系统消息通知。

验收：

- BeeChat 可作为企业内部消息中心底座。
- 外部系统接入有清晰接口与鉴权策略。

## 四、优先级建议

近期优先级：

```text
P0  浏览器视觉验收与 UI 细节修复
P0  文档补齐与仓库整理
P1  前端工程化改造
P1  管理后台与审计详情
P2  PostgreSQL / Redis / 对象存储

```

## 五、风险与处理

| 风险 | 影响 | 处理方式 |
| --- | --- | --- |
| 原 Pencil 源文件丢失 | 设计追溯困难 | 已重建正式 Pencil 源文件 |
| 原生前端继续膨胀 | 后期维护成本升高 | 阶段 3 迁移工程化前端 |
| SQLite 不适合多人生产 | 数据一致性与扩展受限 | 阶段 6 引入 PostgreSQL |
| WebSocket 单实例限制 | 多实例无法广播 | 阶段 6 引入 Redis Pub/Sub |
| 审计权限边界不够细 | 合规风险 | 阶段 4 收敛审计权限 |

3. 进入前端工程化改造方案设计。
4. 决定前端技术栈：React 或 Vue。
