# BeeChat Roadmap

## 一、最终目标

BeeChat 的最终目标是成为面向企业内网私有化部署的协同消息底座，覆盖团队沟通、频道协作、私信、线程、文件、安全审计、组织集成与企业内部系统通知。

长期形态：

```text
企业内网 IM + 协作消息中心 + 审计安全底座 + 企业系统通知入口
```

核心原则：

- 私有化部署优先。
- 数据本地可控。
- 权限与审计优先。
- 先稳定可用，再逐步工程化、平台化。
- 不追求复杂社交功能，优先服务企业协作效率。

## 二、当前所处阶段

当前处于：

```text
阶段 2：设计重建与前端同步
```

已具备：

- 可运行 MVP。
- 登录、频道、私信、消息、线程、文件、搜索、审计、成员管理。
- SQLite 持久化。
- WebSocket 实时刷新。
- 数据库初始化迁移。
- 冒烟测试。
- Pencil 设计源文件重建。
- 需求、设计、接口、部署、迭代文档归档。

当前短板：

- 前端仍是原生 HTML/CSS/JS，长期维护成本较高。
- 缺少管理后台。
- 审计权限与审计详情还不够细。
- 数据库仍是 SQLite，不适合正式多人生产。
- WebSocket 仍是单实例模型。
- 缺少真实浏览器截图验收。

## 十五、关键决策点

近期需要确认：

1. 前端工程化选择 React 还是 Vue。
2. 是否保留当前 Node.js 原生后端，还是后续迁移到 Spring Boot。
3. 生产数据库优先 PostgreSQL 还是 MySQL。
4. 文件存储使用本地磁盘、MinIO 还是其他对象存储。
5. 审计权限是否仅管理员可见。
6. BeeChat 是否作为 BeePilot / BeeWorks 后续消息中心底座。
