BeeChat 需求文档

一、项目定位

BeeChat 是面向企业内网私有化部署的轻量协同聊天系统。第一阶段目标不是替代完整企业 IM，而是先形成可运行、可演示、可迭代的协同聊天底座，支撑频道沟通、私信、线程、文件、搜索、审计与基础权限。

二、目标用户

  • 企业内部员工：日常频道沟通、私信沟通、文件传递。
  • 团队负责人：创建频道、邀请成员、查看协作上下文。
  • 系统管理员：管理频道成员、追踪审计记录、控制基础安全策略。
  • 私有化交付人员：本地部署、数据落盘、验证系统运行状态。

三、当前版本范围

已实现能力

  • 登录与会话恢复。
  • 本地账号与角色区分。
  • 频道列表、创建频道、加入频道。
  • 频道成员查看、邀请、移除。
  • 私信会话。
  • 消息发送、编辑、撤回。
  • 消息分页与加载更早消息。
  • 线程回复。
  • 文件上传与下载。
  • 全局消息搜索。
  • 未读数。
  • WebSocket 实时刷新。
  • 敏感词提示。
  • 操作审计与审计过滤。
  • SQLite 持久化与 JSON 快照兼容。
  • 数据库初始化迁移。
  • Docker 与本地启动脚本。

当前新增设计同步能力

  • 重建 Pencil 多画板设计源文件。
  • 主界面、线程页、登录页、发现频道页、设置页设计规范。
  • 前端视觉风格同步到新版 BeeChat 设计。
  • 发现频道弹层、频道搜索、加入频道交互。
  • 前端动态内容 HTML 转义，降低注入风险。

四、核心业务流程

登录流程

  1. 用户输入账号和密码。
  2. 后端验证账号、密码与角色。
  3. 返回 token、当前用户、工作区信息。
  4. 前端保存会话 token。
  5. 刷新页面后通过 /api/session 恢复登录态。

频道沟通流程

  1. 用户进入默认频道。
  2. 前端加载频道消息第一页。
  3. 用户发送消息。
  4. 后端写入消息、审计、未读状态。
  5. WebSocket 通知客户端刷新。
  6. 用户可编辑、撤回、加载更早消息。

线程回复流程

  1. 用户点击消息的线程入口。
  2. 前端加载该消息下的回复。
  3. 用户在右侧线程面板回复。
  4. 后端按 parentId 关联消息。
  5. 根消息回复数更新。

文件流程

  1. 用户选择文件上传。
  2. 后端保存文件并生成消息。
  3. 文件消息进入频道或私信。
  4. 审计记录写入文件上传行为。
  5. 用户点击文件卡片下载。

审计流程

  1. 登录、消息、文件、成员、频道等关键操作写入审计。
  2. 管理员可在右侧审计面板查看。
  3. 支持按类型过滤。
  4. 敏感消息进入审计提示。

五、权限要求

功能管理员普通用户
登录支持支持
查看已加入频道支持支持
创建频道支持不支持
加入频道支持支持
发送频道消息需为成员需为成员
发送私信支持支持
邀请频道成员支持不支持
移除频道成员支持不支持
查看审计支持当前 MVP 可见，后续需收敛

六、非功能要求

私有化部署

  • 支持本地启动。
  • 支持 Docker Compose。
  • 默认不依赖外部云服务。
  • 数据默认落在项目本地 data/ 目录。

安全要求

  • 密码不得明文存储。
  • 会话 token 需要具备过期策略。
  • 用户输入内容前端渲染前必须转义。
  • 文件上传需要限制目录穿越。
  • 管理操作必须进入审计。

可维护性要求

  • 接口文档、部署文档、需求文档、设计文档、迭代计划分目录归档。
  • 数据库结构变更必须走迁移文件。
  • 冒烟脚本覆盖核心流程。

七、暂不纳入当前版本

  • 企业组织架构同步。
  • 单点登录。
  • 多租户。
  • 移动端原生应用。
  • 完整富文本编辑器。
  • 音视频会议。
  • 大规模分布式 WebSocket。
  • 审计导出与合规报表。

八、验收标准

  • npm run smoke 通过。
  • 登录、发送消息、线程回复、文件上传、搜索、审计、成员管理可用。
  • design/beechat/beechat-main-chat.pen 可被 Pencil 打开并包含五个 BeeChat 画板。
  • 需求、设计、接口、部署、迭代文档路径清晰。
